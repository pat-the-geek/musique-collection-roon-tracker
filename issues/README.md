# Issues Documentation

This directory contains documentation related to specific GitHub issue implementations and fixes.

## Purpose

Store all issue-specific documentation including implementation reports, analysis, fix summaries, and visual comparisons.

## Naming Convention

Files should follow the pattern: `ISSUE-{number}-{description}.md`

Examples:
- `ISSUE-21-IMPLEMENTATION.md` - AI integration implementation
- `ISSUE-47-FIX-SUMMARY.md` - Bug fix summary
- `ISSUE-41-ANALYSIS.md` - Data analysis work

## Contents

This directory contains documentation for issues related to:
- Feature implementations
- Bug fixes
- Code improvements
- Analysis and visualizations

## Guidelines

- **When to use**: Document any work done for a specific GitHub issue
- **What to include**: Implementation details, technical decisions, before/after comparisons, testing results
- **Format**: Markdown files with clear sections and code examples where relevant
- **References**: Link back to the GitHub issue number for context
