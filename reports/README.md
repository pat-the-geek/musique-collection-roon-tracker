# Project Reports

This directory contains project-level reports and documentation.

## Purpose

Store project-wide documentation including release notes, migration guides, analysis reports, and major update summaries.

## Types of Reports

### Release Notes
- Format: `RELEASE-NOTES-v{version}.md`
- Content: Version changes, new features, bug fixes

### Migration Guides
- Format: `MIGRATION-GUIDE.md` or `MIGRATION-{version}.md`
- Content: Instructions for upgrading between major versions

### Analysis Reports
- Format: `ANALYSE-COMPLETE-v{version}.md`, `COHERENCE-CHECK-REPORT.md`
- Content: Project analysis, code quality checks, architecture reviews

### Update Summaries
- Format: `DOCUMENTATION-UPDATE-*.md`, `MISE-A-JOUR-*.txt`
- Content: Documentation updates, reorganization summaries

### Other Reports
- Format: Various (`.md` or `.txt`)
- Content: Timezone fixes, reorganization notes, etc.

## Guidelines

- **When to use**: For project-wide documentation not tied to a specific issue
- **Scope**: Project-level changes, releases, major refactorings
- **Audience**: Developers, users, stakeholders
- **Format**: Clear structure with sections for overview, changes, impact
